# 待确认问题

以下问题没有在项目真相中定义。为不中断实现，当前采用括号内的保守默认值；产品确认后可调整。

- 首次通过邮箱验证码的地址是否自动创建账号？（是；否则空系统中没有可登录账号，且本次范围没有单独的注册流程。）
- 一次性密码的有效期和最大尝试次数是多少？（10 分钟、最多 5 次；超限后该验证码作废。）
- 重发验证码的冷却时间是多少？（同一邮箱 60 秒；冷却期内仍返回相同的通用响应，但不发送邮件。）
- 同一邮箱签发新验证码后，旧验证码是否继续有效？（否；新邮件投递成功后使此前未使用的验证码作废。）
- 会话有效期是多少？（30 天；每次登录创建一个独立会话，退出只撤销当前会话。）
- API 客户端如何携带会话？（使用 `Authorization: Bearer <token>`；本包只交付无界面的 HTTP API，避免替客户端决定 Cookie 策略。）
- 邮箱地址是否区分大小写？（不区分；保存前去除首尾空白并转为小写。）
- 登录响应是否允许暴露账号是否存在？（不允许；请求验证码统一返回 `202`，验证码校验失败统一返回 `401`。）
