- 产品面向个人消费者，同一账号可在多台设备同时登录，用户可以单独退出任意一台设备的会话。
- 登录会话的有效期为 30 天，到期需要重新登录。
- 登录方式只有邮箱加一次性密码，不设长期密码。原因：目标用户不愿再维护一套密码，邮箱本身就是身份介质。
- 账号的唯一标识独立于邮箱，邮箱在完成校验后可以变更。原因：邮箱是联系方式而非身份本体，用户更换邮箱不应丢失账号。
- 一次性密码为 6 位，由数字和字母混合组成，不区分大小写。原因：短且不分大小写，用户才能在手机邮件客户端查看后凭短时记忆在另一台设备上誊写；纯数字在这个长度下组合空间不足，混入字母补足熵。
- 当前用户规模小，部署为单台物理服务器，数据库与键值存储同机，实现不为分布式场景设计。
- 开发与测试环境的邮件投递经由本地 Mailpit（SMTP 1025 端口，HTTP API 8025 端口），不接入外部邮件服务。原因：离线即可开发，测试可经 API 读取实际投递的邮件做断言。
- 邮箱变更只校验新邮箱，不设旧邮箱二次确认。原因：低摩擦优先，登录态即操作授权；双重校验的代价与个人消费场景的风险不匹配，接管风险由用户的设备会话管理自助兜底。
- 邮箱变更不影响既有会话，各设备保持登录。原因：账号标识独立于邮箱，会话挂在账号上；换绑走低摩擦路线，异常会话由用户以踢出设备自助处置。
- 枚举防护以未认证面为界：未认证接口不得暴露邮箱注册状态，已认证接口允许以明确错误提示邮箱是否可用。原因：未认证面无法归因、须防批量探测；已认证面有限流且可追责，模糊响应只牺牲可用性。两个面的差异是有意取舍，不是疏忽。
