# 安全审查结论

- **中危：一次性密码被实现为 6 位纯数字。** 已改为 6 位、同时包含数字和字母的随机码，校验时不区分大小写；保留单次使用、过期和失败次数限制。原因：原搜索空间只有 `10^6`，显著低于项目真相要求，持续在线猜测可能换取 30 天账号会话。
- **中危：发码冷却只按邮箱计算，可通过轮换邮箱无界触发邮件和数据库工作。** 已增加 SQLite 支撑的单实例全局窗口，默认 60 秒最多 10 次，超限返回 `429` 和 `Retry-After`；串行与并发路径均有测试。原因：公开接口不应把任意请求量线性放大为 SMTP 投递和持久写入。
- **低危：已消费和已过期的 challenge 永久保留。** 已在发码前清理离开限流窗口的失效记录，并增加 `created_at` 索引。原因：单机 SQLite 与服务共享磁盘，无界表和索引增长最终会影响认证可用性。
- **低危：SMTP 连接及响应没有截止时间。** 已为完整 SMTP 会话增加默认 10 秒硬超时；超时会关闭 socket、结束 HTTP 请求并回滚尚未投递的 challenge。原因：静默或退化的 SMTP 端点原本可长期占用请求、Promise、socket 和文件描述符。
- **已核查且无需处置：** SQL 均使用参数绑定；OTP 消费与会话创建处于 `BEGIN IMMEDIATE` 事务；会话查询和远程撤销绑定当前账号；Bearer token 仅存哈希；SMTP 地址拒绝 CR/LF；请求体限制为 64 KiB。
