內容結構定義領域聲明檔案內部如何組織。檔案的存在、位置與 frontmatter 欄位由領域聲明規範定義，本規範定義正文。

> 本規範落實 [什麼是專案真相](/zh-hant/argument/what-is-project-truth/) 對真相記錄內容的界定、[自然語言表達應與工作語言一致](/zh-hant/argument/truth-record-work-language/) 的表述原則和 [可推導內容應留在工具層](/zh-hant/argument/derivable-content-in-tool-layer/) 的准入判據，並為 [新鮮度治理](/zh-hant/argument/project-truth-freshness-governance/) 的核查與裁決提供最小內容單元。

## 約定用語

本規範用以下用語表達要求等級：**必須**，實作沒有例外；**應當**，預設遵循，偏離需要有可說明的理由；**可以**，由專案自行決定。否定形式與等級對應：**不得** 是必須級的禁止，**不應** 是應當級的禁止。

## 判斷條目

`TRUTH.md` 正文是一個扁平清單，由判斷條目構成。判斷條目是一個清單項目，以自然語言陳述一個專案目前實際遵循的判斷——規則、約束或取捨。正文從第一個條目開始，沒有引導區。

一個條目 **應當** 只承載一個可獨立裁決的判斷：人裁決它是否仍然成立時，答案應當只有一個。判斷的原因仍影響後續裁決時，**應當** 隨判斷寫在同一條目，讓裁決時原因在場。

```markdown
- 乳牙與恆牙在同一牙位並存時，恆牙覆蓋乳牙顯示。原因：混合牙列期的家長更關注恆牙的萌出進度。
- 牙位編號採用 FDI 記法，介面展示與內部資料保持一致。
- 缺失牙不隱藏牙位，以空位輪廓保留，避免家長把消失的牙位誤讀為資料錯誤。
```

## 入選與退出

寫入正文的判斷 **必須** 滿足專案真相的准入條件：目前專案實際遵循，合理實踐推不出，且專案不接受偏離。複述實作細節的條目 **不應** 寫入。檢驗方法是刪除測試：刪掉一個條目，後續迭代按合理實踐處置仍可接受，它就不屬於正文。

不再遵循的判斷 **應當** 隨確認失效的變更刪除或修訂，修訂經正常變更流程進入版本庫。

## 表述

判斷 **應當** 以工作語言和專案概念表述。能從實作推導的座標，如檔案路徑、目錄名與符號名，**不應** 寫入判斷：實作重構時座標失效，專案概念保持穩定。工程專名可以保留原文。

判斷依賴外部事實成立時，**可以** 隨條目附帶複查線索——日期或可判定的失效條件。線索把外部漂移中可檢驗的部分內化為記錄的一部分；線索的實質隨條目正文表述，不設獨立結構，也不依賴任何習慣用語。需要巡檢按期消費線索時，條目以巡檢標記聲明進入巡檢集合。

## 巡檢標記

需要定期核對外部有效性的判斷條目，**可以** 以巡檢標記開啟內容：`- ` 之後緊隨 `[?] `——方括號包裹單個半形問號，加單個空格。

```markdown
- [?] 風險分級與學會現行指南保持一致。2027-01 核對指南是否更新。
- [?] 服務部署在單台伺服器上，會話可以駐留記憶體。部署拓撲變化時複查。
```

標記聲明一項維護承諾：條目依賴的外部事實需要巡檢定期核對。巡檢集合因此可由機器直接圈定，成員判定不依賴對正文措辭的識別，也不隨工作語言變化。標記表達承諾而非結果：核查是否通過、何時核查過，屬於工具層的過程狀態，**不得** 寫回條目。

標記是條目內容的一部分，參與規範化與內容雜湊：添加或移除標記即條目身份變化，掛靠舊身份的推導與裁決隨之失效。

本規範只定義 `[?]` 一個狀態字元。條目內容以方括號包裹單個字元加空格開頭、而字元未被定義時，構成核查信號。狀態字元表達維護承諾，每個字元對應治理迴路中一處機器可執行的行為差異；內容的分類與分級能從判斷本身推導，屬於工具層快取，**不得** 以狀態字元表達。新的字元經本規範修訂加入。

巡檢標記僅對 `TRUTH.md` 的判斷條目定義。殘留條目的失效由世界驅動，整類屬於巡檢對象，不逐條標記。

## 結構約束

**不得** 為判斷條目設置標題、標識符、類型標籤或核查狀態，以加粗領起充當標題的變體同樣 **不得** 使用。這類內容能從判斷本身推導，屬於工具層快取。巡檢標記不在此列：它聲明的維護承諾無法從判斷推導，由「巡檢標記」一節定義。

條目以行首的 `- `——連字符加單個空格——開啟，至行尾結束，行首不加縮排，**不得** 跨越多行：條目的原始文字邊界因此可由各實作一致復算，正文中不以條目標記開啟的非空行即清單之外的內容。

正文 **不得** 在清單之外放置內容，單行的邊界規則讓違例可由語法檢查直接發現。單行條目也容不下子清單、表格、程式碼區塊等區塊級結構，一個判斷需要區塊級結構才能表達時，正文無法合法承載——它通常應當拆分為多個判斷、收斂為術語條目，或沉入實作的驗證。

## 條目與工具層

條目是正文的最小內容單元。人的裁決與領域間引用以條目為落點，核查信號的落點與處置由治理規範定義；條目在其容器——所屬領域與所在檔案——內以內容雜湊識別，完整落點由容器標識與內容雜湊共同構成，內容變化即身分變化，掛靠其上的推導結果與裁決隨之失效。同一檔案內 **不得** 出現內容完全相同的重複條目，重複可由規範化比較直接判定。快取的儲存形態與失效規則由治理規範定義。

## 術語條目

`GLOSSARY.md` 正文同樣是扁平清單，由術語條目構成。條目以加粗的術語名起頭，隨後給出定義；術語名是專案選定的規範用詞。工具從條目起頭的加粗節點提取術語名；術語名與定義之間的銜接標點隨工作語言的書寫習慣，本規範不作規定。

```markdown
- **恆牙**：替換乳牙後長期使用的牙齒，圖中以實色渲染。
- **乳牙**：兒童期的第一副牙齒，恆牙萌出後被替換。
```

術語條目同樣以行首的 `- ` 開啟，至行尾結束，**不得** 跨越多行；條目內與清單外的區塊級結構約束，與判斷條目相同。同一 `GLOSSARY.md` 內術語名 **必須** 唯一：比較按條目規範化進行，不摺疊大小寫，重名構成機器可判定的衝突。

術語表只作肯定記錄。易混詞的一致性核查由工具從術語名與定義推導並快取，屬於治理規範；個別用詞的拒絕本身承載決策重量時，作為帶原因的判斷條目進入 `TRUTH.md`。

## 殘留條目

`RESIDUE.md` 正文同樣是扁平清單，由殘留條目構成。殘留條目以自然語言陳述一個仍在生效的後果，以及它對後續判斷的要求。一個條目 **應當** 只承載一個可獨立裁決的後果：裁決它是否仍然生效時，答案應當只有一個。

時間或版本線索隨條目正文表述，本規範不為線索規定獨立結構：它為人和 LLM 進入變更記錄提供入口，不需要機器解析。

```markdown
- 牙位表達在 2024-03-17 之前只有單顆牙位，鄰面齲與兩牙各自患齲在此前資料中無法區分；涉及該時段資料的報告需計入此限制。
- v2.0.0 之前的分享連結不含簽章，已分發的舊連結在有效期內仍可存取；收緊存取控制前需先使舊連結失效。
```

條目以工作語言表述。與判斷條目不同，殘留中的歷史座標——當時的版本號、欄位名或格式名——已經凍結，不隨後續重構失效，**可以** 保留以精確定位後果。

「結構約束」與「條目與工具層」兩節的規定同樣適用於殘留條目。

## 待裁決條目

`PENDING.md` 正文同樣是扁平清單，由待裁決條目構成。待裁決條目陳述一個等待裁決的選擇：懸而未決的部分 **應當** 以問句表述，隨後陳述發現者當前採取的處置與理由。問句讓待裁的空間與已經落地的處置在條目內分明，裁決者只需回答問題本身。處置的理由說明在等待裁決期間這樣做為何穩妥——它以合理實踐立論，不預判專案的意圖。

一個條目 **應當** 只承載一個可獨立裁決的問題，多個問題即便同源也拆分為多個條目：捆綁的條目在部分裁決後進退兩難——刪除會丟掉未裁決的問題，保留則繼續以待裁決的口吻陳列已經裁決的部分。

```markdown
- 未註冊的電子郵件請求驗證碼時自動建立帳號嗎？當前按自動建立實作，登入與註冊合一是驗證碼流程的常見形態。
- 會話清單包含已過期的會話嗎？當前只回傳有效會話：清單服務於踢出操作，過期會話沒有可踢出的憑證。
```

條目以工作語言表述。問題與處置的銜接方式隨工作語言的書寫習慣，本規範不作規定；從條目中拆分問題與處置屬於工具層推導。

「結構約束」與「條目與工具層」兩節的規定同樣適用於待裁決條目。

## 範圍之外

殘留與待裁決條目的入選判據與刪除要求由領域聲明規範給出。判斷的寫作手藝——如何凝練表述、原因如何取捨——本規範不作規定。核查節點如何消費判斷條目、巡檢集合與術語一致性信號，以及工具層快取的具體機制，屬於治理規範。投影視圖如何呈現正文內容，屬於編譯規範。
