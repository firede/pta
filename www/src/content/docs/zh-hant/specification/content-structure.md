內容結構定義領域聲明檔案內部如何組織。檔案的存在、位置與 frontmatter 欄位由領域聲明規範定義，本規範定義正文。

> 本規範落實 [什麼是專案真相](/zh-hant/argument/what-is-project-truth/) 對依據部分內容的界定、[自然語言表達應與工作語言一致](/zh-hant/argument/grounding-part-work-language/) 的表述原則和 [可推導內容應留在工具層](/zh-hant/argument/derivable-content-in-tool-layer/) 的准入判據，並為 [新鮮度治理](/zh-hant/argument/project-truth-freshness-governance/) 的核查與裁決提供最小內容單元。

## 約定用語

本規範用以下用語表達要求等級：**必須**，實作沒有例外；**應當**，預設遵循，偏離需要有可說明的理由；**可以**，由專案自行決定。

## 判斷條目

`TRUTH.md` 正文是一個扁平清單，由判斷條目構成。判斷條目是一個清單項目，以自然語言陳述一個專案目前實際遵循的判斷——規則、約束或取捨。正文從第一個條目開始，沒有引導區。

一個條目 **應當** 只承載一個可獨立裁決的判斷：人裁決一個條目是否仍然成立時，答案應當只有一個。判斷的原因仍影響後續裁決時，**應當** 隨判斷寫在同一條目，讓裁決時原因在場。

```markdown
- 乳牙與恆牙在同一牙位並存時，恆牙覆蓋乳牙顯示。原因：混合牙列期的家長更關注恆牙的萌出進度。
- 牙位編號採用 FDI 記法，介面展示與內部資料保持一致。
- 缺失牙不隱藏牙位，以空位輪廓保留，避免家長把消失的牙位誤讀為資料錯誤。
```

## 入選與退出

寫入正文的判斷 **必須** 是目前專案實際遵循、且執行部分難以可靠承擔的內容。執行部分已經清楚表達的細節 **不應** 複寫。檢驗方法是刪除測試：刪掉一個條目，後續迭代的判斷不會因此變化，它就不屬於正文。

不再遵循的判斷 **應當** 隨確認失效的變更刪除或修訂，修訂經正常變更流程進入版本庫。

## 表述

判斷 **應當** 以工作語言和專案概念表述。能從執行部分推導的實作座標，如檔案路徑、目錄名與符號名，**不應** 寫入判斷：實作重構時座標失效，專案概念保持穩定。工程專名可以保留原文。

## 結構約束

**不得** 為判斷條目設置標題、標識符、類型標籤或狀態標記，以加粗領起充當標題的變體同樣 **不得** 使用。這類內容能從判斷本身推導，屬於工具層快取。

條目內 **不應** 使用子清單、表格、程式碼區塊等區塊級結構，正文 **不應** 在清單之外放置段落或標題。清單之外的內容可由語法檢查直接發現；一個判斷需要區塊級結構才能表達時，構成核查信號——它通常應當拆分為多個判斷、收斂為術語條目，或沉入執行部分的驗證。

## 條目與工具層

條目是正文的最小內容單元。核查信號、人的裁決與領域間引用以條目為落點；工具以條目內容的雜湊識別條目，內容變化即身分變化，掛靠其上的推導結果與裁決隨之失效。快取的儲存形態與失效規則由治理規範定義。

## 術語條目

`GLOSSARY.md` 正文同樣是扁平清單，由術語條目構成。條目以加粗的術語名起頭，隨後給出定義；術語名是專案選定的規範用詞。工具從條目起頭的加粗節點提取術語名；術語名與定義之間的銜接標點隨工作語言的書寫習慣，本規範不作規定。

```markdown
- **恆牙**：替換乳牙後長期使用的牙齒，圖中以實色渲染。
- **乳牙**：兒童期的第一副牙齒，恆牙萌出後被替換。
```

術語表只作肯定記錄。易混詞的一致性核查由工具從術語名與定義推導並快取，屬於治理規範；個別用詞的拒絕本身承載決策重量時，作為帶原因的判斷條目進入 `TRUTH.md`。

## 殘留條目

`RESIDUE.md` 正文同樣是扁平清單，由殘留條目構成。殘留條目以自然語言陳述一個仍在生效的後果，以及它對後續判斷的要求。一個條目 **應當** 只承載一個可獨立裁決的後果：裁決它是否仍然生效時，答案應當只有一個。

時間或版本線索隨條目正文表述，本規範不為線索規定獨立結構：它為人和 LLM 進入變更記錄提供入口，不需要機器解析。

```markdown
- 牙位表達在 2024-03-17 之前只有單顆牙位，鄰面齲與兩牙各自患齲在此前資料中無法區分；涉及該時段資料的報告需計入此限制。
- v2.0.0 之前的分享連結不含簽章，已分發的舊連結在有效期內仍可存取；收緊存取控制前需先使舊連結失效。
```

條目以工作語言表述。與判斷條目不同，殘留中的歷史座標——當時的版本號、欄位名或格式名——已經凍結，不隨後續重構失效，**可以** 保留以精確定位後果。

「結構約束」與「條目與工具層」兩節的規定同樣適用於殘留條目。

## 範圍之外

殘留的入選判據與刪除要求由領域聲明規範給出。判斷的寫作手藝——如何凝練表述、原因如何取捨——本規範不作規定。核查節點如何消費判斷條目與術語一致性信號、工具層快取的具體機制，屬於治理規範。投影視圖如何呈現正文內容，屬於編譯規範。
