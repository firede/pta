整合定義架構如何接入既有工程環境：真相入口與生態檔案如何共存，專案級配置放在哪裡，核查節點掛在哪些設施上，共享要滿足什麼最低要求。

> 本規範落實 [新鮮度治理](/zh-hant/argument/project-truth-freshness-governance/) 的節點如何落到工程設施，依 [投影視圖應按需編譯](/zh-hant/argument/projection-view-compiled-on-demand/) 對平行真相的批判界定與生態檔案的關係，並為 [自然語言表達應與工作語言一致](/zh-hant/argument/truth-record-work-language/) 的表述原則提供機器可讀的聲明位置。立論回答為什麼，本規範回答怎麼做。

## 約定用語

本規範用以下用語表達要求等級：**必須**，實作沒有例外；**應當**，預設遵循，偏離需要有可說明的理由；**可以**，由專案自行決定。否定形式與等級對應：**不得** 是必須級的禁止，**不應** 是應當級的禁止。

## 既有工程檔案

AGENTS.md 及同類工程說明檔案是生態產物，由各自的生態約定管轄，不屬於專案真相架構的物件。

工具 **不得** 以這類檔案為專案真相的來源：真相的入口是領域聲明，來源保持唯一。專案 **應當** 讓判斷性內容記錄於領域聲明，工程說明檔案不承載專案真相——承載即形成不受治理的平行材料，與記錄各自漂移。

遷移 **可以** 漸進進行：尚未遷入的內容仍由生態工具照常消費，不因本架構失效；專案按採納節奏收斂重複，工程說明檔案隨遷移逐步瘦身。

專案 **可以** 在工程說明檔案中放置指向專案真相的引用，如 `[專案真相](./TRUTH.md)`，供未接入本架構的工具與 agent 順鏈發現。順鏈讀原文是完整的降級路徑，說明檔案無須為此複述真相內容；接入本架構的消費者經編譯投影獲得真相內容，不經工程說明檔案。

## 專案配置

`pta.toml` 位於倉庫根目錄，是專案級整合配置的唯一入口，語法遵循 TOML 1.0；無法解析的檔案不產生任何配置，並構成核查信號。領域聲明 frontmatter 的 YAML 由 Markdown 生態給定；配置入口獨立採用 TOML，因為自訂配置的類型面寬於聲明欄位，須落在跨實作一致性更強的標準上。

頂層只保留本規範定義的公共欄位：

- `workingLanguage`：聲明專案真相的工作語言。工作語言劃定專案的語義空間，專案 **必須** 顯式聲明，不由內容推斷；版本庫中存在領域聲明而工作語言未聲明，構成核查信號。值為 BCP 47 語言標籤，取語言子標籤，**可以** 附書寫系統子標籤，**不得** 含地區等其餘子標籤：真相記錄是書面物，書寫系統是書面語的構成部分；地區差異下的用詞一致性由專案術語表收斂，不由標籤細分承擔。聲明使表述原則機器可讀，為術語核查與診斷措辭提供依據；不規定工具介面的語言。
- `externalRoots`：列出外置領域聲明根；每項是按標識規範書寫形態給出的目錄路徑。

配置範例：

```toml
workingLanguage = "zh-Hans"
externalRoots = ["packages/web/.pta", "packages/client/.pta"]
```

工具實作的自訂配置 **可以** 放入 `pta.toml`，**必須** 收納於以實作名區分的命名空間之下，如 `[tool.{name}]`，實作之間因此不會爭用同名欄位。實作遇到無法識別的命名空間 **應當** 忽略而非報錯，使多個工具能共存於同一配置。

## 節點掛接

事件觸發的核查節點掛接在工程生命週期的既有設施上——本機鉤子、程式碼審查、持續整合，不新增獨立流程；掛接的具體形態由實作定義。時間觸發的節點以定時任務承載，週期由專案按外部約束的變化速率約定。

強制等級隨位置配置，預設與調整遵循治理規範；本規範不規定任何位置的等級。

## 共享的最低要求

快取與信號在本機使用時，形態完全由實作決定。跨人或跨工具共享時，參與方 **必須** 遵循同一版本的標識規範，使同一內容算出同一標識；**必須** 保全記錄的必備資訊項——信號的落點、類別、證據與來源，處置的方式與物件；鍵的組合序列化與載體格式由參與方約定。

共享儲存可以是檔案、目錄或服務，本規範不作規定。遺失語義按物件區分：投影產物、確認類裁決與處置記錄遵循工具層快取，遺失退回一次推導或裁決成本；已進入迴路、未到終態的信號 **必須** 持久保存至終態，不分來源——事件引發的信號在事件過去後無法重現，遺失即靜默捨棄。進入迴路前的發現屬於工具內部，不受此約束。實作以重放方式恢復時，**必須** 恢復原信號的身分與處置鏈，而非產生無關聯的新信號。

## 範圍之外

候選修訂與核查信號如何呈現給人，是工具的產品設計，本規範不作規定。鉤子的具體命令、巡檢週期的取值，由實作與專案各自決定。跨專案的共享與交付，本規範不作規定。
