集成定义架构如何接入既有工程环境：真相入口与生态文件如何共存，项目级配置放在哪里，核查节点挂在哪些设施上，共享要满足什么最低要求。

> 本规范落实 [新鲜度治理](/argument/project-truth-freshness-governance/) 的节点如何落到工程设施，依 [投影视图应按需编译](/argument/projection-view-compiled-on-demand/) 对平行真相的批判界定与生态文件的关系，并为 [自然语言表达应与工作语言一致](/argument/truth-record-work-language/) 的表述原则提供机器可读的声明位置。立论回答为什么，本规范回答怎么做。

## 约定用语

本规范用以下用语表达要求等级：**必须**，实现没有例外；**应当**，默认遵循，偏离需要有可说明的理由；**可以**，由项目自行决定。否定形式与等级对应：**不得** 是必须级的禁止，**不应** 是应当级的禁止。

## 既有工程文件

AGENTS.md 及同类工程说明文件是生态产物，由各自的生态约定管辖，不属于项目真相架构的对象。

工具 **不得** 以这类文件为项目真相的来源：真相的入口是领域声明，来源保持唯一。项目 **应当** 让判断性内容记录于领域声明，工程说明文件不承载项目真相——承载即形成不受治理的平行材料，与记录各自漂移。

迁移 **可以** 渐进进行：尚未迁入的内容仍由生态工具照常消费，不因本架构失效；项目按采纳节奏收敛重复，工程说明文件随迁移逐步瘦身。

项目 **可以** 在工程说明文件中放置指向项目真相的引用，如 `[项目真相](./TRUTH.md)`，供未接入本架构的工具与 agent 顺链发现。顺链读原文是完整的降级路径，说明文件无须为此复述真相内容；接入本架构的消费者经编译投影获得真相内容，不经工程说明文件。

## 项目配置

`pta.toml` 位于仓库根目录，是项目级集成配置的唯一入口，语法遵循 TOML 1.0；无法解析的文件不产生任何配置，并构成核查信号。领域声明 frontmatter 的 YAML 由 Markdown 生态给定；配置入口独立采用 TOML，因为自定义配置的类型面宽于声明字段，须落在跨实现一致性更强的标准上。

顶层只保留本规范定义的公共字段：

- `workingLanguage`：声明项目真相的工作语言，值为 BCP 47 语言标签。声明使表述原则机器可读，为术语核查与诊断措辞提供依据；不规定工具界面的语言。
- `externalRoots`：列出外置领域声明根；每项是按标识规范书写形态给出的目录路径。

配置示例：

```toml
workingLanguage = "zh-Hans"
externalRoots = ["packages/web/.pta", "packages/client/.pta"]
```

工具实现的自定义配置 **可以** 放入 `pta.toml`，**必须** 收纳于以实现名区分的命名空间之下，如 `[tool.{name}]`，实现之间因此不会争用同名字段。实现遇到无法识别的命名空间 **应当** 忽略而非报错，使多个工具能共存于同一配置。

## 节点挂接

事件触发的核查节点挂接在工程生命周期的既有设施上——本地钩子、代码评审、持续集成，不新增独立流程；挂接的具体形态由实现定义。时间触发的节点以定时任务承载，周期由项目按外部约束的变化速率约定。

强制等级随位置配置，默认与调整遵循治理规范；本规范不规定任何位置的等级。

## 共享的最低要求

缓存与信号在本地使用时，形态完全由实现决定。跨人或跨工具共享时，参与方 **必须** 遵循同一版本的标识规范，使同一内容算出同一标识；**必须** 保全记录的必备信息项——信号的落点、类别、证据与来源，处置的方式与对象；键的组合序列化与载体格式由参与方约定。

共享存储可以是文件、目录或服务，本规范不作规定。丢失语义按对象区分：投影产物、确认类裁决与处置记录遵循工具层缓存，丢失退回一次推导或裁决成本；已进入回路、未到终态的信号 **必须** 持久保存至终态，不分来源——事件引发的信号在事件过去后无法重现，丢失即静默丢弃。进入回路前的发现属于工具内部，不受此约束。实现以重放方式恢复时，**必须** 恢复原信号的身份与处置链，而非产生无关联的新信号。

## 范围之外

候选修订与核查信号如何呈现给人，是工具的产品设计，本规范不作规定。钩子的具体命令、巡检周期的取值，由实现与项目各自决定。跨项目的共享与交付，本规范不作规定。
