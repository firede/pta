内容结构定义领域声明文件内部如何组织。文件的存在、位置与 frontmatter 字段由领域声明规范定义，本规范定义正文。

> 本规范落实 [什么是项目真相](/argument/what-is-project-truth/) 对真相记录内容的界定、[自然语言表达应与工作语言一致](/argument/truth-record-work-language/) 的表述原则和 [可推导内容应留在工具层](/argument/derivable-content-in-tool-layer/) 的准入判据，并为 [新鲜度治理](/argument/project-truth-freshness-governance/) 的核查与裁决提供最小内容单元。

## 约定用语

本规范用以下用语表达要求等级：**必须**，实现没有例外；**应当**，默认遵循，偏离需要有可说明的理由；**可以**，由项目自行决定。否定形式与等级对应：**不得** 是必须级的禁止，**不应** 是应当级的禁止。

## 判断条目

`TRUTH.md` 正文是一个扁平列表，由判断条目构成。判断条目是一个列表项，以自然语言陈述一个项目当前实际遵循的判断——规则、约束或取舍。正文从第一个条目开始，没有引导区。

一个条目 **应当** 只承载一个可独立裁决的判断：人裁决它是否仍然成立时，答案应当只有一个。判断的原因仍影响后续裁决时，**应当** 随判断写在同一条目，让裁决时原因在场。

```markdown
- 乳牙与恒牙在同一牙位并存时，恒牙覆盖乳牙显示。原因：混合牙列期的家长更关注恒牙的萌出进度。
- 牙位编号采用 FDI 记法，界面展示与内部数据保持一致。
- 缺失牙不隐藏牙位，以空位轮廓保留，避免家长把消失的牙位误读为数据错误。
```

## 入选与退出

写入正文的判断 **必须** 满足项目真相的准入条件：当前项目实际遵循，合理实践推不出，且项目不接受偏离。复述实现细节的条目 **不应** 写入。检验方法是删除测试：删掉一个条目，后续迭代按合理实践处置仍可接受，它就不属于正文。

不再遵循的判断 **应当** 随确认失效的变更删除或修订，修订经正常变更流程进入版本库。

## 表述

判断 **应当** 以工作语言和项目概念表述。能从实现推导的坐标，如文件路径、目录名与符号名，**不应** 写入判断：实现重构时坐标失效，项目概念保持稳定。工程专名可以保留原文。

判断依赖外部事实成立时，**可以** 随条目附带复查线索——日期或可判定的失效条件。线索把外部漂移中可检验的部分内化为记录的一部分；线索的实质随条目正文表述，不设独立结构，也不依赖任何习惯用语。需要巡检按期消费线索时，条目以巡检标记声明进入巡检集合。

## 巡检标记

需要定期核对外部有效性的判断条目，**可以** 以巡检标记开启内容：`- ` 之后紧随 `[?] `——方括号包裹单个半角问号，加单个空格。

```markdown
- [?] 风险分级与学会现行指南保持一致。2027-01 核对指南是否更新。
- [?] 服务部署在单台服务器上，会话可以驻留内存。部署拓扑变化时复查。
```

标记声明一项维护承诺：条目依赖的外部事实需要巡检定期核对。巡检集合因此可由机器直接圈定，成员判定不依赖对正文措辞的识别，也不随工作语言变化。标记表达承诺而非结果：核查是否通过、何时核查过，属于工具层的过程状态，**不得** 写回条目。

标记是条目内容的一部分，参与规范化与内容哈希：添加或移除标记即条目身份变化，挂靠旧身份的推导与裁决随之失效。

本规范只定义 `[?]` 一个状态字符。条目内容以方括号包裹单个字符加空格开头、而字符未被定义时，构成核查信号。状态字符表达维护承诺，每个字符对应治理回路中一处机器可执行的行为差异；内容的分类与分级能从判断本身推导，属于工具层缓存，**不得** 以状态字符表达。新的字符经本规范修订加入。

巡检标记仅对 `TRUTH.md` 的判断条目定义。残留条目的失效由世界驱动，整类属于巡检对象，不逐条标记。

## 结构约束

**不得** 为判断条目设置标题、标识符、类型标签或核查状态，以加粗领起充当标题的变体同样 **不得** 使用。这类内容能从判断本身推导，属于工具层缓存。巡检标记不在此列：它声明的维护承诺无法从判断推导，由「巡检标记」一节定义。

条目以行首的 `- `——连字符加单个空格——开启，至行尾结束，行首不加缩进，**不得** 跨越多行：条目的源文本边界因此可由各实现一致复算，正文中不以条目标记开启的非空行即列表之外的内容。

正文 **不得** 在列表之外放置内容，单行的边界规则让违例可由语法检查直接发现。单行条目也容不下子列表、表格、代码块等块级结构，一个判断需要块级结构才能表达时，正文无法合法承载——它通常应当拆分为多个判断、收敛为术语条目，或沉入实现的验证。

## 条目与工具层

条目是正文的最小内容单元。人的裁决与领域间引用以条目为落点，核查信号的落点与处置由治理规范定义；条目在其容器——所属领域与所在文件——内以内容哈希识别，完整落点由容器标识与内容哈希共同构成，内容变化即身份变化，挂靠其上的推导结果与裁决随之失效。同一文件内 **不得** 出现内容完全相同的重复条目，重复可由规范化比较直接判定。缓存的存放形态与失效规则由治理规范定义。

## 术语条目

`GLOSSARY.md` 正文同样是扁平列表，由术语条目构成。条目以加粗的术语名起头，随后给出定义；术语名是项目选定的规范用词。工具从条目起头的加粗节点提取术语名；术语名与定义之间的衔接标点随工作语言的书写习惯，本规范不作规定。

```markdown
- **恒牙**：替换乳牙后长期使用的牙齿，图中以实色渲染。
- **乳牙**：儿童期的第一副牙齿，恒牙萌出后被替换。
```

术语条目同样以行首的 `- ` 开启，至行尾结束，**不得** 跨越多行；条目内与列表外的块级结构约束，与判断条目相同。同一 `GLOSSARY.md` 内术语名 **必须** 唯一：比较按条目规范化进行，不折叠大小写，重名构成机器可判定的冲突。

术语表只作肯定记录。易混词的一致性核查由工具从术语名与定义推导并缓存，属于治理规范；个别用词的拒绝本身承载决策重量时，作为带原因的判断条目进入 `TRUTH.md`。

## 残留条目

`RESIDUE.md` 正文同样是扁平列表，由残留条目构成。残留条目以自然语言陈述一个仍在生效的后果，以及它对后续判断的要求。一个条目 **应当** 只承载一个可独立裁决的后果：裁决它是否仍然生效时，答案应当只有一个。

时间或版本线索随条目正文表述，本规范不为线索规定独立结构：它为人和 LLM 进入变更记录提供入口，不需要机器解析。

```markdown
- 牙位表达在 2024-03-17 之前只有单颗牙位，邻面龋与两牙各自患龋在此前数据中无法区分；涉及该时段数据的报告需计入此限制。
- v2.0.0 之前的分享链接不含签名，已分发的旧链接在有效期内仍可访问；收紧访问控制前需先使旧链接失效。
```

条目以工作语言表述。与判断条目不同，残留中的历史坐标——当时的版本号、字段名或格式名——已经冻结，不随后续重构失效，**可以** 保留以精确定位后果。

「结构约束」与「条目与工具层」两节的规定同样适用于残留条目。

## 待裁决条目

`PENDING.md` 正文同样是扁平列表，由待裁决条目构成。待裁决条目陈述一个等待裁决的选择：悬而未决的部分 **应当** 以问句表述，随后陈述发现者当前采取的处置与理由。问句让待裁的空间与已经落地的处置在条目内分明，裁决者只需回答问题本身。处置的理由说明在等待裁决期间这样做为何稳妥——它以合理实践立论，不预判项目的意图。

一个条目 **应当** 只承载一个可独立裁决的问题，多个问题即便同源也拆分为多个条目：捆绑的条目在部分裁决后进退两难——删除会丢掉未裁决的问题，保留则继续以待裁决的口吻陈列已经裁决的部分。

```markdown
- 未注册的邮箱请求验证码时自动创建账号吗？当前按自动创建实现，登录与注册合一是验证码流程的常见形态。
- 会话列表包含已过期的会话吗？当前只返回有效会话：列表服务于踢出操作，过期会话没有可踢出的凭证。
```

条目以工作语言表述。问题与处置的衔接方式随工作语言的书写习惯，本规范不作规定；从条目中拆分问题与处置属于工具层推导。

「结构约束」与「条目与工具层」两节的规定同样适用于待裁决条目。

## 范围之外

残留与待裁决条目的入选判据与删除要求由领域声明规范给出。判断的写作手艺——如何凝练表述、原因如何取舍——本规范不作规定。核查节点如何消费判断条目、巡检集合与术语一致性信号，以及工具层缓存的具体机制，属于治理规范。投影视图如何呈现正文内容，属于编译规范。
