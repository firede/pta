内容结构定义领域声明文件内部如何组织。文件的存在、位置与 frontmatter 字段由领域声明规范定义，本规范定义正文。

> 本规范落实 [什么是项目真相](/argument/what-is-project-truth/) 对依据部分内容的界定、[自然语言表达应与工作语言一致](/argument/grounding-part-work-language/) 的表述原则和 [可推导内容应留在工具层](/argument/derivable-content-in-tool-layer/) 的准入判据，并为 [新鲜度治理](/argument/project-truth-freshness-governance/) 的核查与裁决提供最小内容单元。

## 约定用语

本规范用以下用语表达要求等级：**必须**，实现没有例外；**应当**，默认遵循，偏离需要有可说明的理由；**可以**，由项目自行决定。

## 判断条目

`TRUTH.md` 正文是一个扁平列表，由判断条目构成。判断条目是一个列表项，以自然语言陈述一个项目当前实际遵循的判断——规则、约束或取舍。正文从第一个条目开始，没有引导区。

一个条目 **应当** 只承载一个可独立裁决的判断：人裁决一个条目是否仍然成立时，答案应当只有一个。判断的原因仍影响后续裁决时，**应当** 随判断写在同一条目，让裁决时原因在场。

```markdown
- 乳牙与恒牙在同一牙位并存时，恒牙覆盖乳牙显示。原因：混合牙列期的家长更关注恒牙的萌出进度。
- 牙位编号采用 FDI 记法，界面展示与内部数据保持一致。
- 缺失牙不隐藏牙位，以空位轮廓保留，避免家长把消失的牙位误读为数据错误。
```

## 入选与退出

写入正文的判断 **必须** 是当前项目实际遵循、且执行部分难以可靠承担的内容。执行部分已经清楚表达的细节 **不应** 复写。检验方法是删除测试：删掉一个条目，后续迭代的判断不会因此变化，它就不属于正文。

不再遵循的判断 **应当** 随确认失效的变更删除或修订，修订经正常变更流程进入版本库。

## 表述

判断 **应当** 以工作语言和项目概念表述。能从执行部分推导的实现坐标，如文件路径、目录名与符号名，**不应** 写入判断：实现重构时坐标失效，项目概念保持稳定。工程专名可以保留原文。

## 结构约束

**不得** 为判断条目设置标题、标识符、类型标签或状态标记，以加粗领起充当标题的变体同样 **不得** 使用。这类内容能从判断本身推导，属于工具层缓存。

条目内 **不应** 使用子列表、表格、代码块等块级结构，正文 **不应** 在列表之外放置段落或标题。列表之外的内容可由语法检查直接发现；一个判断需要块级结构才能表达时，构成核查信号——它通常应当拆分为多个判断、收敛为术语条目，或沉入执行部分的验证。

## 条目与工具层

条目是正文的最小内容单元。核查信号、人的裁决与领域间引用以条目为落点；工具以条目内容的哈希识别条目，内容变化即身份变化，挂靠其上的推导结果与裁决随之失效。缓存的存放形态与失效规则由治理规范定义。

## 术语条目

`GLOSSARY.md` 正文同样是扁平列表，由术语条目构成。条目以加粗的术语名起头，随后给出定义；术语名是项目选定的规范用词。工具从条目起头的加粗节点提取术语名；术语名与定义之间的衔接标点随工作语言的书写习惯，本规范不作规定。

```markdown
- **恒牙**：替换乳牙后长期使用的牙齿，图中以实色渲染。
- **乳牙**：儿童期的第一副牙齿，恒牙萌出后被替换。
```

术语表只作肯定记录。易混词的一致性核查由工具从术语名与定义推导并缓存，属于治理规范；个别用词的拒绝本身承载决策重量时，作为带原因的判断条目进入 `TRUTH.md`。

## 范围之外

`RESIDUE.md` 的条目要求由领域声明规范给出。判断的写作手艺——如何凝练表述、原因如何取舍——属于指南。核查节点如何消费判断条目与术语一致性信号、工具层缓存的具体机制，属于治理规范。投影视图如何呈现正文内容，属于编译规范。
