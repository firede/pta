The Project Truth Architecture specification defines how project truth is organized, maintained, used, integrated with tools, and reused across projects. Specifications are derived from the arguments, and each specification declares the arguments it depends on.

<CanonEntries volume="specification" />

The specification layer consists of six documents: domain declaration, content structure, governance, compilation, identity, and integration.
