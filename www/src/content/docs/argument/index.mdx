立论记录项目真相架构成立所依赖的核心判断。

它说明这些判断解决什么问题、为什么成立、边界在哪里。使用者可以定期核查这些判断在当下是否仍然成立，并基于仍然成立的判断推导出适合自身场景的项目真相架构。
