- core 的行为以规范文档为唯一依据：规范未规定处不私铸语义，实现暴露的规范缺口以核查信号回流规范修订。原因：实现是规范的检验器，工具私铸语义会掩盖规范缺口，让分叉实现失去共同界面。
- core 是纯库，不含命令行、网络与 LLM 调用；CLI、Web 等消费界面在核外构建。原因：查看与管理界面会各自演化，回路地板必须可被任何界面复用。
- core 的运行时依赖仅限所依循公开标准的解析器（TOML 1.0、YAML 1.2），其余保持零依赖；条目提取按单行规则以行操作实现，不引入 Markdown 解析器。原因：钉住标准的格式交给带跨实现测试套件的标准解析器，自造子集才是方言风险；单行条目规则的价值是提取可被各实现一致复算，Markdown 解析器则会引入方言。
- 领域发现的范围是版本库视角的工作树：已跟踪与未被忽略的未跟踪内容，被忽略的路径不构成领域。原因：领域声明与版本库树同构是规范前提，文件系统全扫会把构建产物与依赖内容误认为领域。
