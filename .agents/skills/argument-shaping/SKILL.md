name: argument-shaping
description: 将模糊的想法，通过多轮追问塑形成对象明确、判断清晰、理由成立的立论（Argument）。适用于用户想讨论观点、原则、设计判断、标准依据时。
---

把用户的想法塑形成一篇可写作的立论。

立论要素：

- 对象：这篇立论讨论什么。
- 判断：对这个对象提出什么主张。
- 理由：为什么这个判断成立。
- 边界：这个判断适用于什么范围，不适用于什么范围。
- 反对意见：可能的质疑是什么，如何回应。

工作方式：

1. 维护当前已确认的立论状态，包括对象、判断、理由、边界、反对意见。
2. 找出当前最大的缺口。
3. 每次只问一个关键问题。
4. 每个问题给出一个基于用户已说内容的候选表述，供用户确认、修正或否定。
5. 不要过早完成文章。
6. 不要替用户发明核心判断。
7. 当对象、判断、理由、边界、反对意见足够清晰时，经用户确认后，输出完整立论。

完成条件：

- 对象明确。
- 判断清晰。
- 理由能够支撑判断。
- 边界不会导致明显误用。
- 关键反对意见已有回应。
- 用户确认可以进入成稿。

输出规范：

- 聊天中输出完整立论时，只输出文章正文，不带 YAML front matter。
- 只有在用户要求创建、写入或输出立论文件时，才应用文件规范。
- 立论文件放置在 `www/src/content/docs/argument/` 目录下。
- 文件名使用 `{slug}.md`，`slug` 为立论的简短标识符，只能包含小写字母、数字和短横线。
- 立论文件的 YAML front matter 中，`title`、`description` 是必填字段；`draft`、`dependsOn` 是可选字段。
- 初次创建立论文件时，默认写入 `draft: true`。
- 只有当当前立论依赖其他立论成立时，才写入 `dependsOn`；不要编造依赖。
- 输出立论文章时，参考 [立论模板](./TEMPLATE.md) 的结构。
